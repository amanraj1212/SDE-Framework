2) CORS Security in Django
CORS / API exposure
Right now, unless you added a CORS library, Django returns no CORS headers, which means:
Browsers will block cross‑origin AJAX calls by default.
Your APIs are effectively callable only from:
Same origin (your own dashboard), or
Non‑browser clients (Postman, curl).
For a real deployment, you’d typically:
Install django-cors-headers.
In settings.py configure tightly, e.g.:
CORS_ALLOWED_ORIGINS = [    "https://factory-dashboard.company.com",]CORS_ALLOW_CREDENTIALS = False
Do not set CORS_ALLOW_ALL_ORIGINS = True in production.
This ensures only your official dashboard domain can call the /myapp/... APIs from a browser.
