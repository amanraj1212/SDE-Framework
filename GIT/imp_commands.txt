5) When you run git tag aman, Git creates a tag named "aman" on your current 
commit. A tag is just like a label or bookmark. It marks that specific commit 
so you can easily return to it later. It does not change your name, it does 
not change your branch, and it does not affect future commits. It simply 
remembers that particular commit with the name "aman".

When you run git checkout aman, Git moves you to the commit that was marked 
by the tag "aman". Your project files will now look exactly like they were at 
that time. However, you are not on your normal branch anymore â€” you are in 
something called a "detached HEAD" state. This means you are just viewing 
that old commit. It does not delete any commits, and your branch history 
remains unchanged.
